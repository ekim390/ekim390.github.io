Missile Defense

Programming for Hardware/Software Systems - ECE 2035

The goal of this individual project was to recreate the Atari Missile Command game using the ARM mbed microcontroller, an LED screen, pushbuttons, and an accelerometer. It required knowledge of C and object-oriented programming. This project resulted in a game shown on the LED screen. The user or player uses the buttons to determine the starting level of the game on the initialization screen. When picked, the player can begin playing the game with the goal of destroying the missiles before the missiles destroy all the buildings. Movement was controlled by the accelerometer. It detects if the board is tilted left or right and moves the rocket in that direction, having a greater speed the more the board is tilted. To attack the missiles, the pushbutton is used. The pushbutton triggers the rocket to send a laser to attack the missile. If all buildings are destroyed, the player is sent to the game over screen. If they manage to succeed, the player wins. I created a struct for a doubly linked list and implemented code that created missiles and laser beams, used a list to keep track of structures, and erased the buildings and sprite when hit by the missiles and the missiles when hit by the laser beams.

  • Software/Skills learned: C, Object-Oriented Programming
  • Hardware: Mbed Microcontroller, uLCD, Buttons, Accelerometer, Speaker
  • Github Repository: Missile Defense Code

Photo Gallery Application

Cloud Computing - ECE 4150

This is a individual lab that resulted in the creation of a Photo Gallery using AWS. The goal of this individual project was to create a Photo Gallery application composed of albums and photos using 2 variants: SQL and NoSQL to store records of the photos using AWS. The SQL variant: The records of albums and photos are maintained in a MySQL database instance on Amazon RDS. The NoSQL variant: The records of photos are held in an AWS DynamoDB table managed through NoSQL code. I used python to create the application, which uses the Flask web framework and is deployed on an Amazon EC2 instance. I used HTML to code the web interface of the photo gallery. I used python to implement a method for user management and authentication as well as updating, deleting, and adding photos and albums to the application.

  • Software/Skills learned: Python, HTML, CSS, Javascript, Amazon Web Services AWS
  • Github Repository: Photo Gallery Code

Tower Defense Game

Objects & Design - CS 2340

This Tower Defense game is called: The Scarlet Tower.

The Scarlet Tower is a group project built using JavaFX, building upon good software design practices and software architecture. The game consists of a Base that the player needs to protect to win, 4 types of enemies whose goal are to get to the end of the path and attack the Base, and 3 types of Towers that the player can purchase between rounds.

The game begins when the user presses the start button and enters a name and difficulty level in the player configuration. From there, the user can really start the game. There is a shop button that enters into a new window where players have 3 different towers to choose from that will attack enemies and protect the Base. Tower 1 attacks only, Tower 2 drains the health from the enemies, and Tower 3 earns money for attacking enemies. After picking a tower, the tower can be placed anywhere besides the path, base, and other towers. After placing all the tower, the player can start the round. During the round, random enemies come out and begin walking on the path at different speeds. At this time, the towers can attack. These attacks are implemented through visual lines that act as lasers. When enemies get into the Tower’s proximity, the closest one is attacked, lowering the enemy health. If the enemy manages to get all the way to the Base with enough health, it rams into the base, disappearing and lowering the health of the Base. If the player manages to win all 4 rounds, the player wins, triggering the win screen. If not, the player loses and the game over screen triggers.

  • Software: Java, JavaFX
  • Github Repository: Tower Defense Game Code

Mad Hatter Thespian Robot

ECE Design Fundamentals - ECE 3872

This group project is the creation of a robot inspired by one of the Alice in Wonderland characters. This robot’s base design is a green hat that is based on the mad hatter. The green hat moves up and down when speaking its line, inspired from the movie. While “speaking” or a series of beeps for words, the robot will also have a yellow led pattern in front of a cup that holds the dormouse who moves up and down as well during the line. This robot sits on top of a box that holds the PCB, raspberry pi, parts such as the servos, leds, and speaker, and wires that combined: control the robot.

This project took around 9-12 weeks to complete and test robot. First several weeks for ideation and planning (designing robot, thinking of parts needed, and software architecture), next 3 weeks to compile components of the electrical, mechanical, and software and test separately (worked on software components so pi connection/code), and the last 6 weeks involved building the robot & compiling all components so that they work together. I mostly worked on the software and Raspberry Pi subsystem, learning how to concurrently perform each action with good timing.

The original goals for the project were to have a series of red and green leds around the hat, the top of the hat opening and closing, and the speaker playing a wav file for the line. However, we realized that the current power we have in the system wouldn’t be enough to power the green leds and there were no red leds available when we were building the system. Therefore, the team collectively chose to use yellow leds instead and place them in front of the coffee cup to be displayed better. Although it would have been better to choose red leds seeing how the yellow leds are not as visible in the demo. The top of the hat opening and closing did not seem feasible because we already had a linear servo motor moving up and down to extend and flatten the hat like it’s moving. So, instead of having this action, we decided to add a coffee cup on the side of the hat that contains the dormouse (another character from the movie) moving the lid up and down.

The main objectives I had during this project was to learn how to use the Raspberry Pi Zero W 2, solder headers onto the Pi, and strengthen my skills in soldering when adding headers and connecting wires and in laser cutting. We successfully were able to create a robot that runs all the actions concurrently through python script in the Raspberry Pi. The major actions I had to do were laser cut wood for box, develop code to control actions through headless connection on the raspberry pi, solder headers to raspberry pi & figure out clean pin assignments.

  • Software: Python (Raspberry Pi GPIO library, headless connection)
  • Hardware: Raspberry Pi Zero W 2, PCB, servo motors, leds, analog speaker
  • Skills: soldering, documentation, laser cutting, PCB fabrication
